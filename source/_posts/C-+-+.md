# 基本类型

对于宽字符型来说按照如下方式定义

```cpp
typedef short int wchar_t;
```

也就是说宽字符型和短整型使用同样大小的空间。

## 分类

整型：布尔、字符、整数
算术类型：整型、浮点型
用户自定义类型：类、枚举
内置类型：基本类型、指针、引用

## 布尔类型

布尔值取true或者是false，bool可以转换为int类型而不会发生信息丢失，转换之后true变为1，false变为0。但是当整数类型向bool转换的时候，会发生窄化转换，通常来说非0值转换为true，0值转换为false。

如果想通过使用`{}`将int类型转换为bool可以使用以下方法。

```cpp
void f(int i) {
   bool b {i != 0};
}
```

计算过程：在算术逻辑表达式或者位逻辑表达式中bool会自动转换为int，在结果需要bool的情况下会转换回bool

bool与指针的隐式转换：非空指针对应true，nullptr对应false。

## 字符类型

char：用来实现C++的字符集，程序文本。前可加符号限定符。在不同实现中可能会将有无符号完全等效，一般情况要将这三种情况分开。
wchar_t：用于存放Unicode，依赖于实现。
char16_t：UTF-16存放16位字符集。
char32_t：UTF-32存放32位字符集。

问题：
1. 有些嵌入式处理器无法按照字节读取内存，因此char实现为占用4个字节。

### 将char转换为int

将char转换为int类型时符号的问题，由于C++支持使用补码的硬件，因此有符号数从-127~127，但是问题只会在大于127的时候出现。

### 指向char的指针

虽然char的行为与`signed char`或者`unsigned char`类似，但是进行指针赋值的时候不能混用。

### 字符字面值常量

在单引号内部的单字符被称为字面值常量，它可以隐式的转换为当前机器对应字符集的整数。

### 转义字符

十六进制：反斜线x后跟位数不限的十六进制数，原理同八进制数。

将多个字符放到单引号中，已经过时，不要使用，它的类型时int。

## 整数类型

不加符号修饰的int是带符号的。

### 字面量后缀

U: `unsigned int`
L: `long int`

后缀可以多个字符组合起来使用。当字面值常量没有后缀的时候，会按照`int` `long int` `long long int `取能表示的最小的范围。

对于某个数字，在不同位数的机器上会有不同的类型，应该使用后缀来避免对实现的依赖性，提高移植性。

# 声明

在使用某一个名字的时候必须要对其进行声明，同时可以对其进行定义。

不可以重复定义，不可以定义上下不一致。

```cpp
   int count;
   int count; //重复定义

   extern int error_number;
   extern short error_number; //上下定义不一致
```

定义会显式的赋值，对于类型、别名、模板、函数、常量定义之后值不可变，非const类型的初始值会在稍后改变，

## 声明的结构

```cpp
const char* kings[] = {'a', 'b', 'c'};
```

定义了一个数组，这个数组的每个元素都是`const char*`类型，

### 作用域

1. 局部作用域：在程序块中，从声明处开始，到块结束的地方，对于函数以及lambda表达式来说，它们的参数也是在程序块中的局部作用域

2. 类作用域：从类开始到类结束中的名字，称为成员名字或者是类成员名字。

3. 名字空间作用域：与类作用域类似，但是名字空间作用域可以被其他访问单元访问。

4. 全局作用域：可以将全局作用域看作是一个特殊的名字空间，这个名字空间是贯穿整个程序的。

5. 语句作用域：定义在`for while if switch`中的一些变量从定义开始到语句范围结束的。

6. 函数作用域：在函数中从声明开始到函数结束。

如果以上作用域形成了嵌套，同名定义内层会对外层进行屏蔽。我们可以使用作用域解析运算符来访问外层定义。

```cpp
int a = 1;
void foo(void) {
   int a = 2;   //定义了内层的变量，此时屏蔽了块外的变量。
   ::a;         //a = 1
   a;            //a = 2
}
```

### 初始化

在初始状态下被赋予的值，初始化器有以下四种形式，只有第一个形式在使用的时候不受任何限制，但是其C11刚刚提出的，因此在老代码中后面三种还是常见。

```cpp
int a1 {1};
int a2 = {1};
int a3 = 1;
int a4(1);
```

使用`{}`进行初始化被称为列表初始化，可以防止窄化转换。

窄化转换：两个整型，前者无法在不丢失信息的情况下转化为后者，那么就不会进行转化。对于浮点型有一样的道理。

整型无法转化为浮点型，浮点型也无法转化为整型。

如果我们使用auto推断类型的时候不应该使用列表初始化，**在使用auto时要使用=**

#### 缺少初始化器

静态对象：全局变量、名字空间变量、局部static变量、static成员。

动态对象（堆对象）：局部变量和自由存储上的变量。

对于静态对象如果没有初始化器会自动使用`{}`进行初始化。动态对象除非在自定的构造函数中否则不会执行默认的初始化。

### 推断类型

#### auto

使用auto推断出初始化器的类型，当类型非常复杂的时候auto会非常有用，在较小的作用域内首选auto。

换句话说，如果在很大的定义域中使用auto无助于错误的定位。

我们应该为推断出的类型添加修饰符，表达式的类型不会一直是引用类型。

```cpp
void g(int& v) {
   auto x = v;    //int x
   auto& y = v;   //int& y
}
```

## 声明的顺序

看下面的例子

```cpp
struct S;

extern S a;    //S类型的a变量
S f();         //返回值是S的函数
void f(S);     //函数中使用S作为形式参数的类型
S* h(S*);      //指向S类型的指针
```

我们声明了一个叫S的结构，但是就第一行来说我们不知道结构体S中到底有什么，也就是我们只进行了声明没有进行定义。

后面四条语句都是在还不知道S中到底有什么的情况下进行声明的，因此我们必须先定义好S之后才能使用后续的声明。

# 指针

于C语言中的指针一样`T*`指向T类型的指针，或者可以通过取地址运算符将指针指向对应类型。

指针的具体实现与机器的寻址机制同步，不同机器的寻址方式不一样，因此有可能按照字节寻址，从字中抽取出字节，很少能够直接寻址二进制位。

能够独立分配且用内置指针指向的最小对象是char类型，bool类型至少和char类型占用同样多。当想把值存的更紧密的时候可以使用位逻辑操作、结构中的位域或bitset。

## void*

这里是void基本类型的一个应用例子，在底层代码中不知道对象的确切类型的时候会用void\*来指向：指向未知类型对象的指针。但是如果在系统的上层出现了void\*那么就需要考虑设计上的问题。

void*可以指向除函数指针、指向类成员的指针外的所有类型指针。

直接对void\*进行操作会导致不安全还会导致编译器报错，因为在寻址的时候是按照字节进行寻址，但是如果无法知道应该寻址多少个字节就会导致错误，因此我们应该将void\*强制转换为其他类型进行操作。但是强制类型转换之后会同样会导致数据被截断。

主要用途：
1. 当无法知道对象的类型的时候，我们需要向函数传递指向该对象的指针。
2. 用于从函数返回未知类型的对象，同样使用该类型对象的时候仍然需要对其进行强制类型转换。

## nullptr

表示空指针，对于任何类型的空指针都用这个表示，并非每个类型都有一个空指针。

在nullptr引入之前使用0代表空指针，任何指针都无法将内存分配到全0位上。在C语言中通常会用`NULL`来表示空指针，但是C语言的实现是`(void*)0`，也就是一个`void*`类型，但是这个类型无法分配到其他类型上。

当重载函数既可以接受整数与指针的时候使用nullptr可读性更高。

## 指针与const

对于常量有两种：
1. constexpr：编译时求值。
2. const：指定在作用域内值不会发生改变。主要是要保证接口的不可修改性。

我们无法对const的对象进行赋值因此它必须进行初始化。

修饰方法如下：

```cpp
const char* c;
char* const c;
```

这两种写法可以看const靠着哪个近一点，第一个靠着char近，因此const修饰的时char，也就是说char是一个常量，那么第一句是指向常量char的指针。

第二句中const靠着指针近，因此修饰的是指针，也就是指针是常量，指针不可变，常量指针指向了char类型。

# 引用

如果T表示类型比如`int`之类的，那么指针的定义方式是`T* ptr`，引用就是将\*换成&，也就是`T& int rr`。

对于某一个对象来说，一个引用相当于取某个对象，将这个对象的指针赋给引用，但是在使用引用的时候会自动将引用解引。

```cpp
int var {1};
int& rr = var;
rr++;
```

虽然是rr自增，但是相当于rr的中的地址解引之后再自增，地址是var的地址，它解引之后是1，自增就是2，所以最后var的值会变成2。

此时交换两个数的函数就变的简单起来了。当函数声明中形式参数是引用类型的时候，实际参数只要填入对应的类型即可，函数中会自动将其指针传入，并且会自动解引用。

对于不想函数修改实参的时候我们可以将参数设置为`const int& a`这样就可以避免函数修改实参。

```cpp
void swap(int& a, int& b) {
   int temp = a;
   a = b;
   b = temp;
}
int main() {
   int a = 1;
   int b = 2;
   swap(a, b); //a = 2, b = 1
}
```

从可读性的角度来说，应该尽量避免使用函数来改变实参的值，应该通过返回值来显式表示。

对于一个普通的左值引用来说，它必须和某个具体的对象进行绑定，也就是说，当声明一个引用的时候就必须对其进行初始化，同时绑定的对象不能是一个字面值常量，因为我们无法对一个字面值常量取其地址。也就是说下面的形式非法。

对于返回值来说我们也可以将其设置为引用类型，因为如果需要返回的值过于庞大，那么拷贝数据需要很大开销，这时不如将返回值设置为引用。

```cpp
char& c = 'c';    //非法
```

但有些时候我们需要这么做，比如在向函数传递参数的时候，我们可以将引用声明为const引用此时`const char& c = 'c'`就是合法的了。

右值引用，这个引用与const左值引用有相同的赋值方法，通过使用两个&表示右值引用`char&&`但是const左值引用强调的是我们不会更改这个引用，但是优质引用意味着我们可以肯定会更改这个引用，因此我们不会定义const的右值引用。在某些场合如果拷贝数据开销过大的时候我们可以使用右值引用来移动这些数据，可以避免拷贝的过大开销。

# 数组

有T类型，那么`T[size]`表示有size个T类型元素组成的，索引范围从0到size-1。使用下标访问符访问数组中的元素，数组名会隐式的向指针转换，因此也可以使用指针访问数组。

**越界访问数组会造成未定义行为，可能会引发严重错误，C++中不检查且无法保证越界访问的发生。**

数组大小在定义的时候就确定，数组是C++中表示内存对象序列的最基本的方式。

数组没有内置的拷贝操作，因此不可以用一个数组给另一个数组赋值，但是可以用字符串字面值常量初始化字符数组。

## 字符数组

使用`\0`作为char数组的终止符，这是从C语言延续下来的传统，一些标准库也是建立在这个用法之上，比如：strcpy()、strcmp()，而`char*`和`const char*`指向以`\0`结尾的字符序列。

## 数组初始化器

可以使用`{}`初始化数组，在没有给定size的时候数组会根据初始化器自动计算数组大小。如果同时提供size和初始化器的时候，如果初始化器中的数量多余给定的size会报错，如果少于给定size，剩余的部分会默认用0填充。

## 字符串字面值常量

字符串字面值常量会比表现出来的多一个字符，也就是`\0`，因为从C语言流传下来的传统是字符串是用`\0`来结尾的。因此`"bill"`的大小是5。

在C11之前的代码，字符串字面值常量可以给`char*`赋值也就是非常量，但是现代代码不可以，同时将字符串字面值常量赋值给`const char*`之后就不可以对其进行修改了。如果希望能够继续修改，可以将其赋值到一个数组中去。

这个东西是静态分配的，函数返回字符串字面值常量很安全。

```cpp
const char* p = "this is a string";
const char* q = "this is a string";

if (p == q) {
   std::cout << "one";
}
```

以上语句会判断对于不同指针指向相同的字符串时，这两个指针是否位同一个指针，答案是依赖于实现。如果输出了one那么就表示两个指针的地址都是一样的，因此也就是两个指针相同。

允许在字符串中出现空字符也就是`\0`，但是大多数程序会忽略空字符后面的内容，例如很多标准库函数`strcmp()`，因为这些函数都是一开空字符工作的。如果出现了一个`Jens\000Munk`会被当做时`Jens`。

## 原生字符串字面值常量（原始字符串）

当我们需要输入很多反斜线或者双引号的时候，我们同样需要反斜线来进行转义，这样就会造成混乱尤其是在书写正则表达式的时候。

在原始字符串中的所有字符串怎么输入的就怎么表示。同样如果在其中输入了换行，也会在最终的打印结果中出现

```cpp
std::string s = R"(\w\\w)";
```

需要注意的是原生字符串不仅有这一种分割放上，只要在括号两侧的符号相同就可以，在处理正则表达式的时候非常有用。

```cpp
std::string p = R"*&^(\\as\\f123\45\\asdf\a\sdf\)*&^";
std::cout << p;
```

这里的分隔符是`*&^`（随便输入的），这样也就可以说明原始字符串的分割方式。

## 数组中的指针

数组名可以看作是指向数组中第一个元素的指针，在使用指针遍历数组的时候，因为C++不检查越界，所以当指针指向数组外的元素并不会报错，但是如果要对数组外的元素进行存取都是未定义的行为，C++不能保证发生什么。

数组名向数组中首元素的指针转换在C语言风格的代码中广泛应用于向函数传递参数。也就是向函数参数传递的是数组的首元素指针，会发生从数组到指针的显式或隐式转换，因为函数无法接受整个数组。但是不存在指针向数组的不管是显式还是隐式转换。

这也就出现一个问题，一旦数组向指针转换之后，数组的大小就会丢失，一些程序中我们需要显式的传入数组的size。

## 遍历数组

可以使用指针加索引值来访问或者是下标访问，以下语句第一个函数使用的是下标遍历数组，第二个是通过指针遍历数组。

```cpp
void fi(char v[]) {
   for (int i = 0; v[i] = 0; i++) {
      use(v[i]);
   }
}
void fp(char* v) {
   for (char* p = v; *p != 0; p++) {
      use(*p);
   }
}
```

这两个版本生成的代码应该是一样的（对于现代的编译器来说），对于数组a和数组范围内的整数j有下列式子成立。

```cpp
a[j] == *(&a[0] + j) == *(a + j) == *(j + a) == j[a];
```

对于`a[j] == j[a]`在实际代码中用处不多，上面的规则非常底层，并不适用于vector或者是array。

# 结构

数组中有相同类型的元素，结构中可以有不同类型的元素。我们可以使用点运算符成语进行赋值，也可以用`{}`进行初始化，要按照结构中的顺序进行初始化。

如果两个类内部的类型一模一样，也不能等价类与基本类型也不能等价。

## 布局

结构成员按照声明的顺序依次存放，但是由于在结构中存在不同类型的元素，因此可能会出现某个元素占不满，但是剩下的空间存放某个元素又存不下的问题，因此结构内部可能会有一些空隙，这样可以便于机器高效的处理这些数据，这样的机器我们说它们对其的很好。

如果不是需要精心设计以节省内存的话，建议结构内的成员排列按照可读性的角度来排列。

## 结构的名字

只要结构名字出现，无须等到声明完成就可以使用，比如在结构中定义一个指向该结构的指针。但是由于结构的定义还没有完成，因此在结构内部不能出现同名结构。如果需要多个结构内部相互引用的话需要提前声明以下结构。

```cpp
struct List;

struct Link {
   Link* pre;
   Link* suc;
   List* member_of;
   int data;
};
struct List {
   Link* head;
}
```

虽然没有定义List，但是在前面已经声明过List因此只要在Link中不使用List也是可以的。

C++中允许有两个同名的struct元素和非struct元素，但是鉴于可读性应避免使用同名实体。

## 与类的关系

结构也是一种类，但是它的成员默认是public的，这一点与类相反，struct同样也可以包含成员函数。

# 语句

1. 分号：只有一个分号的也是一条语句被称为空语句。
2. 块（复合语句）：用花括号括起来，在块中声明的变量作用域直到块结束。
3. 声明：没有赋值或者过程调用的语句，赋值和函数调用被称为表达式。
4. for初始化语句：要么是声明要么是表达式语句都以分号结束。
5. for初始化声明：必须是未初始化变量的声明。
6. try语句块：用于异常处理。

## 声明作为语句

如果在没有为一个值找到合适的变量的时候最好不要声明它，防止未初始化变量造成的程序错误。如果一个变量没有设置初始化器大多是情况是为了等待用户的输入

## switch

因为在case中的语句不会确保执行到，因此在case中初始化语句，但是int这类的声明语句不必初始化，因此在编译器并不会检测到这一点。

## for

```cpp
for (int i = 0; i < 10 i++) {
   //受控语句
}

int i = 0;
while(i < 10) {
   i++
}
```

for的基本用法如上，在括号内的语句被称为初始化语句以及表达式语句，for语句的语句也即是块中的语句被称为受控语句，下面的while与上面的for相同，但是如果我们需要最终的索引值的话我们就不应该在for中初始化一个索引变量，因为在for中变量的作用域只到for结束。

### 范围for语句

允许我们依次访问指定范围内的每个元素。

```cpp
for (int x: v) {

}
```

可以表示为对于范围v中的所有元素x或程序从头到尾访问v中的所有元素。这要求范围v的定义中必须要有迭代器。也即是我们调用`v.begin()`和`v.end()`应该得到一个范围，对于内置数组`T v[N]`会令v是begin令v+N是end。

范围for循环非常简单，无法处理同时遍历多个序列的功能，如果想要做到这一点应该使用普通的for循环。

## goto

在大多数情况下用不到，但是由其他程序生成C++代码的时候非常重要。goto的限制是不能跳过初始化器以及跳入到异常处理程序。

最常用的用法是当有多层循环嵌套的时候，我们使用break只会跳出最内层的循环，因此当需要跳出所有循环的时候我们应该使用goto直接跳出，这个时候不会造成任何麻烦。

```cpp
int main() {
   while (true) {
      while (true) {
         while (true) {
            goto found;
         }
      }
   }
   found:
      //foo();
}
```

# 类

## 基础

一个类就是一个用户自定义类型，这个类型中有属性以及方法。通过public提供接口通过private实现内部细节。类的成员默认是private的。

## 成员函数

一些函数与我们定义的类之间没有关系，但是我们可以通过将这些函数声明为成员函数来对类中的属性进行操作。我们在类中声明可以在其他地方定义，也正因为这样，在其他地方可能会有同名函数，我们需要在前面加上访问语法。

```cpp
class Date {
   int d_, m_, y_;
   void init_date(int dd, int mm, int yy);
};

void Date::init_data(int dd, int mm, int yy) {
   d_ = dd;
   m_ = mm;
   y_ = yy;
}
```

在成员函数的声明中，我们可以无需使用访问函数，直接访问类中的其他成员。 

## 默认拷贝

当我们需要声明一个类型之后，如果在此之前还有同类型的对象，可以使用这个对象对新对象进行初始化，这个默认过程是复制拷贝的。

```cpp
class Date{};
Date d1 = my_date;
Date d2 {my_date};
```

当然有些拷贝消耗巨大我们可以为其提供更为恰当的方式。

## 访问控制

使用struct的时候所有的成员默认都是公有的，但是类默认是私有的，当类中的成员只想让其成员函数访问时可以将他们设为私有（private），而公有的（public）可是当作是类的接口对外暴露，通过接口来对类进行操作。

同时这里就会出现另一个问题，如果我们的私有的函数之后成员函数能够时候的话，当我们将属性设置为私有的时候初始化函数就显得非常重要了。

我们也要注意到，对成员设置私有保护只能防止误操作，而不能防止恶意操作。

在一般使用的时候通常会将类的公有成员放在前面以强调接口。

## 构造函数

如果某个函数就是专门用来完成对象初始化工作的，那么这个函数就被称为构造函数，构造函数的本质就是给定一个类型的值。

```cpp
class Date {
public:
   Date(int dd, int mm, int yy);
private:
   int d, m, y;
}

Date today = Data(19, 1, 2023);
Date maxd(31, 12, 99);

Date today = Date {19, 1, 2023};
Date maxd {31, 12, 99};
```

构造函数最显著的特征就是函数名和类名相同，并且函数没有返回值。通常会使用`{}`来使用构造函数，这样可以保证初始化的一致性。

可以通过提供多个构造函数来为不同的值提供初始化方法，构造函数的重构方式和普通函数相同，只要函数的参数明显不同就可以选择正确的版本。

### 默认构造函数

如果没有为对象提供一个初始值那么它应该具有什么值。就拿vector来说如果不为它提供任何值，他会默认使用空元素，这是因为vector拥有默认的构造函数。

当一个构造函数不接受任何参数的时候我们称之为默认构造函数。

## 类内函数定义（内联函数）

如果一个成员函数在类内被定义，那么这个函数会被当做内联函数，只有那些不常修改的小函数会在类内进行定义。
