# 抽象数据类型

抽象之后我们没有必要知道到底发生了什么，我们只需要知道这样做能产生什么样的结果就行。

将程序分割为不同的模块，通过调用不同的**模块**运行程序，这样做有很多好处：

1. 可以将修改限定到特定的模块中，在修改的时候可以只修改不同的模块，有利于多人同时进行程序的编写。
2. 调试小程序的时候会比调试大程序更加简单，并且可以大大减少编译的时间。
3. 不同的模块相互之间是透明的，只需要模块能实现功能就可以。但同时在进行模块话编写的时候一定要注意全局变量的问题。

*抽象数据类型*，就是进行数学抽象，并且不暴露具体的模块实现细节。

# 基本概念

数据：数据是信息的载体。

数据元素：是信息的基本单位，我们对信息的操作是以数据元素为单位的。

数据对象：拥有相同性质的数据元素的集合。

数据类型：

1. 原子类型：值不可以再进行分割的数据类型。比如整型、浮点型等。

2. 结构类型：值可以再进行分割的数据类型。比如C语言中的结构体

3. 抽象数据类型：抽象数据类型的组织以及相关的操作。

数据结构：数据元素之间的关系。

## 数据结构三要素

数据元素之间的关系。

1. 逻辑结构
    逻辑结构就是数据从逻辑关系上进行描述，逻辑结构是独立于计算机的，与数据元素的存储没有关系。比如链表虽然在逻辑上是连续的，第一个元素后面是第二个，但是**实际**在计算机中并不一定是连续的。
2. 物理结构
    由于物理结构与逻辑结构是相互独立的，物理结构也就是数据在计算机中实际存储的方式。物理结构依赖于编程语言的实现。包括顺序、链式、索引，hash。
3. 数据的运算
    运算是依附于物理结构的对数据元素的操作方式。我们通过编程语言对数据结构的物理结构进行操作达成我们所希望的运算。

## 算法

算法的五个基本特性

1. 有穷性：算法在进行有穷步之后一定会结束，并且每一步都会在有限的时间内完成。
2. 确定性：同一个算法对于相同的输入一定会得到相同的输出。
3. 可行性：算法的所有操作都可以以已经实现的基本操作步骤实现。
4. 输入：有0或者多个输入。
5. 输出：有1或多个输出。

优秀算法的特征

1. 正确性：算法可以正确的解决问题。
2. 可读性：算法便于人类理解。
3. 健壮性：对于错误的输入可以进行相应的处理，对于不合法的输入可以进行相应的提示与拒绝
4. 效率与低存储需求：对于相同的需求，执行越快的算法，需求存储空间越少的算法越优秀。比如在排序算法中，冒泡算法比快排需求的额外存储空间要少，但是大多数情况下快排比冒泡要快得多。算法在不同的应用场景下各有优劣。

### 算法的效率

通过时间复杂度和空间复杂度来进行划分，通常使用大O表示法进行表示。通常考察数量级，对于一万个数据与处理一万零一个数据几乎没有差别。空间复杂度如果使用的额外空间为常数级，称算法原地工作。

# 线性表

一般就是一个从１到N的表，大小是N。

- 空表：大小为0的表。

- 前驱：某个元素的前一个元素，第一个元素没有前驱。

- 后继：某个元素的后一个元素，最后一个元素没有后继。

ADT是一种数学的抽象，ADT本身不定义操作，当我们规定只能从最后一个位置添加删除元素，那么就成了一个操作受限的线性表（栈）。

## 线性表的基本操作

以下只是对线性表的基本操作，其他的复杂的操作都可以由基本操作来实现。只是说明线性表的基本操作，**并不是标准的操作接口**。

```c
//初始化一个线性表
init_list(&L);
//求表长
length(L);
//定位元素位置
localelem(L, e);
//获取给定元素位置
getelem(L, i);
//向线性表插入元素
list_insert(&L, i, e);
//删除第i个元素的并且值通过e返回
list_delete(&L, i, &e);
//打印线性表
print_list(L);
//线性表判空
is_empty(L);
//销毁线性表并且释放线性表所占用的空间。
destroy_list(&L);
```

## 顺序表

物理结构，一片连续的内存中的元素。

### 优点

1. 顺序表的逻辑结构和物理结构是相同的。通常使用数组来进行存储。需要注意的是，数组中从0开始存储，而顺序表中是从次序1开始记录。并且顺序表的存储密度很大，所有占用的空间都被用来进行元素的存储。链表实现的话由于需要在元素的结构体中增加用于记录指向下一个元素的指针，因此对比顺序表来说，链表的存储密度要小。
2. 使用数组进行存储，可以通过计算直接算出需要索引的元素的位置。换句话说：我们需要索引第3个元素的时候只需要将顺序表的起始节点加上3即可，对比链表的话，我们同样需要索引第3个元素的时候我们无法通过计算或者其他只需要操作一次的方法就得到需要索引的元素的位置。我们必须先找到第一个，之后顺着第一个找到第二个，进而找到第三个。所以，顺序表的最大优点就是随机索引，我们想要得到某个位置的元素的时候可以直接得到。

### 局限性

1. 我们需要首先对表的最大值进行估计，这个过程通常会估计的大一些，但是这样就会造成空间的浪费。
2. 但是如果使用数组来实现的话，某些操作的执行时间代价巨大，比如插入和删除。在第一个位置插入或删除元素需要将所有的元素都前移或者后移。两种操作的时间复杂度为`O(N)`，平均来看仍然需要移动一半的元素。并且如果通过N次相继插入元素来创建一个表将会需要二次的时间。

对于第一个问题，我们可以使用动态分配的顺序表，如果一开始分配的空间不够，我们可以开辟一块比初始空间更大的空间对原空间进行替换。需要注意的是，动态分配和静态分配都是线性表，都可以进行随机存取。

第二个问题，如果线性表需要频繁的插入删除元素，可以改用链表。顺序表适用于插入删除不那么频繁的情况。因此我们需要在一开始就要考虑数据应该如何进行组织，考虑数据会被怎样应用。对于线性和链式两种结构，后面的排序算法也会有所限制，有些排序算法需要进行随机索引，因此只能使用线性表进行实现。

## 链表

如果我们的表需要频繁的插入删除操作，可以使用链式存储结构来实现。下述定义了一个链表的节点。我们一个结构体中包含一个数据域和一个指向下一个链表节点的指针。

```c
typedef struct node{
    elemType value;
    struct node *next;
} node, *linkedList;
```

在增删改查的时候我们可以仅仅通过改变指针的指向就可以对整个表进行操作。但是同样如果需要索引的时候由于链表节点在内存中并非连续存储，因此我们只能通过第一个节点来依次遍历直到找到我们需要的节点。也就是说我们失去了顺序表的随机索引的优点。同时，由于在顺序表中一个节点只有需要存储的元素，但在链表中还需要保存指向下一个节点的指针，存储密度会有所降低。
链表适合需要频繁修改表的情况，修改的时间复杂度为`O(1)`但是索引的复杂度会变为`O(N)`

**问题**

1. 如果需要在链表的最后插入一个元素，那么仍然需要遍历整个链表，而遍历的时间复杂度为`O(N)`，此时单链表并不比顺序表省时。
2. 由于在插入或者删除的时候仍然需要前一个节点的地址，但前一个节点的地址保存在在前一个节点中，递归可以发现，要找到插入位置需要修改前一个节点，所以要索引找到前一个节点的地址。
**设计上的问题：**
1. 如果无法在表前显性的插入一个元素。也就是没有头节点的情况，如果进行插入
2. 从表的开头删除元素会造成表的丢失。
3. 如果删除表可能会产生一个无头指针，也就是无法索引到这个指针。
**解决方法：**
上面的问题可以通过加入一个表头（header）或者哑节点（dummy node）。当然这个表头可加可不加，根据个人习惯。但是加上程序更加清晰。

### 单链表

头指针表示单链表，这种方式我们可以声明一个指针，这个指针指向NULL的时候表示是一个空表。但是这种表示方式在处理第一个和其他节点的时候会有不同，因此我们可以使用带头节点的方式进行处理单链表

在加入头节点之后我们可以统一第一个与其他节点的操作，我们将头指针始终指向头节点，头节点中不存储有效数据，但是可以存储表长。不仅我们在处理第一个位置与其他位置的时候可以得到统一，而且由于头指针始终指向头节点，并且始终非空，因此我们在处理空表与非空表的时候也得到了统一。

1. 头插法建立单链表：将每个新的节点都插入到链表的开头，这样我们就建立了一个与插入顺序相反的单链表。头插法的特点就是简单，我们无需其他的操作，直接找到链表就可以进行插入。但是头插法建立的单链表与插入顺序是相反的，如果想得到与插入顺序相同的链表应该使用尾插法。
2. 尾插法建立单链表：在使用尾插法的时候由于我们是在链表的末尾进行插入元素，由于链表无法随机访问，因此我们应当设置一个尾指针以指示尾部在哪里，这样我们就无需每次在进行尾插法的时候都从头索引一遍单链表。由于我们有了尾指针来指示尾部的位置，因此找到尾部的时间复杂度依然是$O(1)$，与头插法相同。

**节点的插入**

这里非常重要，因为对于链表来说插入与删除如果颠倒次序就会造成指针的丢失。举个例子：在单链表中我们在A节点与B节点之间插入X节点，由于B节点连接到A节点，因此在A节点中的指针域存放的是B节点的地址，由于我们要将X节点插入到A节点之后，也就是将A节点中的指针域内的地址修改为X节点的地址。但是B节点的地址我们仍然需要，并且需要将地址存放在X节点的指针域中，因此我们必须要先将B节点的位置交给X节点，再将A节点中指针域内的地址修改为X节点的地址。因此正确的顺序就是先将A节点中指针域内的地址赋值给X节点内的指针域，再将A节点中指针域内的地址修改为X节点的地址。

就像我们要交换两个变量的值，在逻辑上我们可以简单的描述为让A=B，再让B=A，但是在实际操作的时候我们要找到一个temp变量来临时存放A或B中的值，因为在赋值的时候A或B其中的某一个变量的值必定已经修改了，但是我们又需要修改前的值，就要找个临时的地方来存储这个值：temp=A;A=B;B=temp;。

### 双链表

为了解决单链表只能向后索引不能向前索引的问题，引入双链表，就是在每个节点中增加一个指向前一个节点的指针prior。

单链表在插入值的时候为了避免指针丢失，需要提前保存指针，由于单链表中的插入删除比较简单，只要稍加注意就没有问题，但是双链表中由于A节点保存着后面节点的信息，如果先释放了这个指针会造成无法索引到后面的B节点，同时B节点也指向A节点，这个指针也就丢失了。因此我们要保证A节点中指向B节点的指针域最后释放即可，否则一定会造成链表丢失。

在我们删除X节点的时候由于我们将指针指向了待删除节点，只需要将X节点中需要保留的数据（指向A、B节点的指针），之后也不要忘记将X节点占用的内存释放，防止造成内存泄漏。

另一种删除节点的方法：删除节点在逻辑上来说就是一个数据元素的消失，我们通过在一个结构体中保存其他的信息实现了链式结构，但是我么使用的是存储在数据域中的内容，因此我们通过将待删除元素与其后的节点交换数据域的方式，删除待删除节点之后的节点同样在逻辑上也能达成删除元素的效果，不过实际我们是通过删除待删除节点之后的元素并且交换二者的值完成的。

### 基本操作

1. IsEmpty操作：判断这个表是否为空表

```C
int IsEmpty(list l) {
    return l->next == NULL;
}
```

这个函数很简单，只要判断，这个节点的`next`指针为`NULL`那么就证明这个节点后面没有节点了，也就是这个节点为最后一个节点。
2. Find操作：用来返回某个元素在这个链表中的位置。

```c
position Find(elementType value, list l) {
    p->next;
}
```

我们定义一个位置，这个位置为链表的下一个节点，当这个节点不是最后一个节点，，并且这个节点的值不为要查找的值的时候，我们获取本节点的下一个节点，并且继续上述的判断。
只要上述判断不满足的时候，也就是这个节点的值等于我们要搜索的值的时候我们就返回这个节点，这样我们就能查找到某个值在链表中的位置。

# 树

## 二叉树

### 定义

每个节点最多只有两个子树，**严格区分次序**。

区分：

1. 度为二：一个节点只能有两个子树，

2. 二叉树：可以有零个子树。

### 性质

### 完全二叉树

高为$h$且有$n$个节点的完全二叉树与高为$h$的满二叉树在1~n的节点一一对应。

当 $i<n/2$ 时为分支节点
当 $i>n/2$ 时为叶子节点。

### 满二叉树

高$h$的满二叉树有$2^n-1$个节点，除最底层外，每个节点都有两个分支。

节点i有双亲时，双亲为$i/2$

节点i有子树时，左子树为$2i$右子树为$2i+1$

## 二叉树的应用

### 带权路径长度（WPL）

路径长度与权重的乘积。树的WPL为所有结点的WPL和。

### 霍夫曼树、霍夫曼编码

当WPL最小的时候所得的树为霍夫曼树也称为最优二叉树。

可变长度编码：每个字符使用不同长度的编码。

固定长度编码：每个字符使用相同长度的编码。

前缀编码：没有任何一个编码是另一个编码的前缀，读取的时候不会产生歧义

通过霍夫曼树来构造霍夫曼编码是非常自然的。

### 构造霍夫曼树

将n个结点作为n个空结点的树，并且使之成为森林F。将森林中的权重最小的两个结点作为孩子结点添加到另一个结点中，并添加到森林F中。重复以上操作，直至森林F中只有一颗树。这棵树就是霍夫曼树。

# 图

图G是由顶点集V和边集E组成，记作$G=(V,E)$用$｜V｜$表示顶点的个数，用$|E|$表示边的条数。

## 概念

线性表可以是空表，树可以是空树，但是图不可以是空图。

1. 有向图：当E为有向边（弧）的有限集合时，图G就是有向图。弧是顶点的有序对，记作$<v,w>$，称为从v到w到弧，也称：v邻接到w其中vw是顶点，v是弧尾，w是弧头。
2. 无向图：当E是无向边的有限集合（边）时，图G就是无向图。边是顶底的无序对，记作$(v,w)$，vw互为邻接点，边(v,w)依附于w和v，边(v,w)和v与w相关联。
3. 简单图：不存在重复的边，不存在顶点到自身的边。
4. 多重图：两顶点之间的边可以大于一条，并且顶点可以通过一条边与自身相连。
5. 有向完全图：在任意两个顶点上有方向相反的两条弧。｜E｜的取值范围为$n(n-1)$
6. 无向完全图：任意两个顶点之间都有一条边连接。|E|的取值范围为$n(n-1)/2$
7. 子图：当$v'$是$v$的子集$E'$是$E$的子集时，我么就称$G'$是$G$的子图，同时当$V(G')=V(G)$时，我们称$G'$是$G$的生成子图。
8. 连通、连通图、连通分量：在无向图中，如果顶点vw之间有路径，那么我们就称两顶点*连通*，如果无向图中所有顶点都是连通的，那么我们就成无向图是*连通图*。否则就是非连通图。其中，无向图中的*极大连通子图*称为*连通分量*。如果一个图有n个顶点，当边数小于n-1时，这个图就是非连通图。
9. 强连通、强连通图、强连通分量：如果有向图的两个顶点vw从v到w和从w到v都有路径，那么这两个顶点就是强连通的，如果有向图中所有顶点都是强连通的那么这个有向图就是强连通图，其中极大强连通子图称为强连通分量。在有n个节点的强连通图中，最少需要n条边构成回路才能构成强连通图。
**在有向图中讨论强连通性，无向图中讨论连通性**
10. 生成树、生成森林：连通图中包含全部节点的极小连通子图称为生成树。有n个节点的连通图其生成树有n-1条边。对于生成树来说，砍掉一条边就成了非连通图，加上一条边就出现环路。在非连通图中连通分量构成的生成树组成了非连通图的生成森林。
11. 极大连通子图：该连通子图包含其所有的边。
12. 极小连通子图：既要保持图连通，又要是的边数最少。
13. 顶点的度：在无向图中，顶点v的度是指，依附于顶底v的条数，记作$TD(v)$对于有n的顶点e条边的无向图G $\sum_{i=1}^nTD(v_i)=2e$，也就是无向图中所有顶点的度等于边树和的两倍。每条边都与两个顶点相关连。
14. 入度、出度： 在有向图中，顶点v的度分为入度和出度。所有以v为终点的有向边的数目为入度，记作$ID(v)$，所有以v为起点的有向边的数目为出度，记作$OD(v)$。顶点v的度$TD(v)=OD(v)+ID(v)$。对于有n个顶点e条边度有向图，$\sum_{i=1}^nID(v)=\sum_{i=1}^nOD(v)=e$也就是有向图中所有顶点的入度和等于出度和等于边树，这是因为每条有向遍都有一个起点和一个终点。
15. 边的权和网：在一个图中每条边上可以有一个具有某种意义的数值，称为**权值**。边上带权值的图称为带权图，也叫**网**。
16. 稠密图和稀疏图：边树很少的图称为稀疏图，边数很多的图称为稠密图。二者的概念是相对的，并且概念比较模糊。一般当图G满足$|E|<|V|log|V|$时可以将图G视为稀疏图。
17. 路径、路径长度、回路：从一个顶点到另一个顶点之间的一条路径是指经过的节点的集合。路径上边的数目称为路径长度。当第一个节点和最后一个节点相同的时候这条路径称为回路或者环。当图G有n个节点并且边的数目大于n-1时，这个图就一定有一个环。
18. 简单路径、简单回路：在一个路径序列中，顶点不出现重复的路径称为简单路径。在一条回路中，除了起点和终点之外的其他路径序列中不出现重复的回路称为简单回路。
19. 距离：从一个顶点到另一个顶点如果存在最短路径，那么这个路径就是这两个顶点之间的距离。如果两顶点之间的路径不存在，那么称这两个顶点之间的距离为$∞$。
20. 有向树：如果有一个顶底的入度为0其余顶点的入度均为1的有向图称为有向树。

## 图的存储以及基本操作

### 邻接矩阵法

使用一个一维数组存储存储顶点的信息，使用一个二维数组存储边的信息。存储顶点之间邻接关系的二维数组就是邻接矩阵。当第$i$节点和第$j$个节点相连的时候二维数组$A[i][j]=1$否则等于0。对于带权图来说，可以在邻接矩阵中存储权重信息，如果不相连则使用$∞$或者0表示。

注意：
1. 简单应用时可以只使用二维数组进行存储。省略顶点的信息。
2. 当邻接矩阵仅仅表示路径是否存在的时候可以使用bool类型的变量进行表示，减少邻接矩阵的存储空间。
3. 无向图的邻接矩阵是一个对称矩阵，在超大规模的应用中可以使用压缩存储。
4.  邻接表的空间复杂度为$O(n^2)$，n为图G的顶点个数$|V|$

#### 存储特点

1. 无向图的邻接矩阵一定是一个对称矩阵，并且矩阵唯一。在使用的时候一般采用压缩存储的方式。
2. 对于无向图来说，邻接矩阵的第i行或者i列中，非零元素的个数正好时节点i的度。
3. 对于有向图来说，邻接矩阵中第i行非0元素的个数就是顶点i的入度，第i列非0元素的个数就是顶点i的出度。
4. 使用邻接矩阵存储图的时候很容易确定任意两个节点是否相连，但是如果要确定有每个顶点有多少条边需要按行、列对每个元素进行检索，时间代价巨大。
5. 稠密图适合使用邻接矩阵进行存储，稀疏图会有非常多没有使用的存储空间，会造成浪费。
6. 如果一个图G的邻接矩阵为$A$则$A^n$的元素$A^n[i][j]$等于顶点i到顶点j的长度为n的路径的条数。离散数学中有证明。

### 邻接表法

为了解决存储的图为稀疏矩阵时浪费大量空间的问题。

有向图存储：节点使用顺序存储，每个元素是一条单链表。第i条单链表表示依附于第i个节点的边。这个单链表称为第i个节点的边表。

无向图存储：节点依然使用顺序存储，与无向图相同。但是链表则是以vi为尾的弧。

综上：在邻接表中有两种数据结构，一种是以顺序存储表示的所有顶点。另一个则是以单链表表示的与顶点相临的顶点。

#### 存储特点

1. 如果G是无向图，那么所需存储空间为$O(|V|+2|E|)$，因为每个节点都会重复记录与其相邻的顶点。如果G是有向图，所需存储空间为$O(|V|+|E|)$。
2. 对于稀疏矩阵来说邻接表法可以非常有效的节省存储空间，因为在邻接矩阵中，不管是否存在边，都会被记录，在邻接表中，只记录相邻的顶点，会极大的降低存储开销。
3. 在邻接表中，可以非常容易读取给定的顶点的所有邻边。但是在邻接矩阵中需要进行遍历才能确定，时间复杂度为$O(n)$但是如果需要判断两顶点是否相连，邻接表需要则需要遍历才能确定，而邻接矩阵可以直接查找确定。因为邻接表的提出就是为了解决稀疏矩阵存储的浪费空间问题。在实际应用过程中，使用邻接表进行判断两节点是否相连时，时间复杂度也不会太高，但是不如邻接矩阵快，因为邻接矩阵是随机读取。
4. 有向图的邻接表存储是存储的出度信息，要确定一个节点的出度信息只需要遍历单链表即可，但是入度信息则需要遍历整个邻接表。同样，我们也可以将邻接表存储入度信息构成逆邻接表，加速入度的搜索。
5. 图的邻接表表示并不唯一，因为输入顶点的次序可以不同。

### 十字链表法

是有向图的一种存储结构，顶点需要一个节点来存储，弧也需要一个节点来存储。同时需要注意，顶点节点之间的存储是通过顺序存储来保存的。

弧节点：分别有尾域、头域、弧头链域、弧尾链域和信息域。尾域和头域分别指示弧尾和弧头在图中的位置。尾链域和头链域分别指向弧尾相同和弧头相同的下一条弧。信息域则存储弧的相关信息。

顶点节点：数据域用于存储数据，弧头域和弧尾域分别指向弧头或弧尾的第一个弧节点。

#### 特点

十字链表存储了节点的入度和出度信息，因为既可以找到以vi为弧头的弧，也可以找到以vi为弧尾的弧。虽然每个有向图都有不同的十字链表，主要是因为一个节点的入度出度信息通过链表存储，链表有次序，但是节点的入度出度没有次序。但是一个十字链表只能确定出一个有向图。

### 邻接多重表

是无向图的另一种链式存储方式。与十字链表相似，需要分为边节点和顶点节点。与十字链表相同，顶点节点也是通过顺序存储来实现的。

边节点：标志域，用于标记该条边是否被搜索过。ivex和jvex用于表示该条边依附的两个节点在图中的位置。ilink和jlink用于指向下一条依附于i节点和v节点的边。信息域用来存储边的相关信息。

顶点节点：数据域，用于存储数据，firstedge域，用于指向第一条连接到节点的边。

## 图的遍历

从某一个节点出发，按照某种算法将图的所有节点都遍历一遍，并且只能遍历一次。需要注意的是树也是一种特殊的图，因此对于图的遍历方法同样可以适用于树。

图相比于树来说，遍历的复杂程度要高很多，图可以相当于树的兄弟节点之间还是连接的，因此有可能会重复遍历，因此，我们对图的遍历，需要记录下遍历过的节点。因此我们可以设置一个辅助数组visted[]来记录某个节点是否已经遍历过。

---

**遍历算法**

### 广度有限搜索（BFS）

首先访问一个节点，然后访问与这个节点相邻的节点。之后在依次访问上一次相邻的但是没有访问过的节点。拿一棵树来说，广度优先遍历就是从树的根节点先横向遍历，遍历完成之后进入下一次继续横向遍历。

通过借助辅助队列来看：树的根节点入队，搜索根节点的下一层节点，根节点出对，下一层节点依次入队。节点出队，同时索引这个节点的子节点，将其入入队。重复以上队列的操作，直到辅助队列为空为止。并且广度优先遍历的序列顺序与二叉树的层次遍历顺序完全相同。

#### BFS算法性能

时间复杂度：对于邻接表来说，每个节点都需要入队一次，因此时间复杂度为$O(|V|)$在搜索任意节点的邻接点时，每条边至少搜索一次，因此时间复杂度为$O(|E|)$，因此总的时间复杂度为$O(|V|+|E|)$。对于邻接矩阵来说，查找一个节点都需要所需的时间复杂度都是$O(V)$因此，总的时间复杂度为$O(|V^2|)$。

空间复杂度：对于邻接表和邻接矩阵来说，BFS都需要一个辅助队列，最坏的情况所有节点都需要入队一次，因此时间复杂度为$O(|V|)$

#### BSF求解单源最短路径问题

由于BFS算法是由近及远进行遍历的，因此如果想找到一条从$v~u$的最短路径只要从u开始进行BFS遍历到v，所得的路径便是最短路径。

#### 广度优先生成树

在BFS的过程中我们可以得到一课广度优先生成树。需要注意的是，由于一个图只能确定一个邻接矩阵，因此通过邻接矩阵创建处理的广度优先生成树是唯一的。但是一个图并不能对应一个邻接表，因此通过邻接表创建的广度优先生成树并不是唯一的。

### 深度优先搜索（DFS）

深度优先搜索类似与树的先序遍历，找到一个节点之后要尽可能沿着这个节点所在的分支搜索。基本思路就是，首先访问一个节点，找到与这个节点相邻的另一个节点，并且沿着这个刚找到的节点继续遍历未遍历的节点。重复上述过程，直到所有的节点都被遍历。

#### DFS性能分析

1. 时间复杂度：取决于存储结构，对于邻接矩阵来说，遍历v个节点，每个节点的邻接点时间复杂度为$O(|V|)$，因此对于邻接表来说，总的时间复杂度为$O(v^2)$。对于邻接表来说，每个节点遍历的时间复杂度为$O(|E|)$，因此总的时间复杂度为$O(|E|+|V|)$

#### 深度优先生成树、深度优先生成森林

对于连通图来说，通过DFS可以得到一颗树，当然，对于非连通图来说得到的就是深度优先生成森林。

### 图的遍历和连通性

对于无向图来说，如果图是连通的，那么通过一次遍历就可以将全部的节点都遍历。否则就不能通过一次遍历遍历所有节点。对于有向图来说，如果图是连通的，那么从初始点到每个节点都有路径，否则就是不连通的。

对于无向图来说，调用BFS和DFS的次数就是无向图的连通分量数。对于有向图来说，由于连通的有向图有强连通分量和非强连通分量，通过一次BFS或者DFS无法对非强连通分量进行一次遍历，尽管图是连通的。

## 图的应用

### 最小生成树

连通图的生成树包含所有节点，并且在保证连通的情况下包含的边最少。对于生成树来说，去掉一条边就成了非连通图，增加一条边就会产生一个回路。对于带权图来说，在图G的生成树集合中权重和最小的那颗生成树称为图G的最小生成树。

性质：

1. 最小生成树不唯一，最小生成树的树形也不唯一。当图的各条边权重不同时，最小生成树就是唯一的。当无向连通图G的边树比节点数少1的时候，图G就是一棵最小生成树。
2. 最小生成树的权值是唯一的且是最小的，虽然对应的最小生成树树形可能不同。
3. 最小生成树的边数等于顶点数减1。

#### 最小生成树算法

算法都是基于贪婪算法策略，包括Prim算法和Kruskal算法

1. Prim算法：在连通图中选择一个节点，将节点加入到最小生成树中，此时的树只有一个节点，找到与这个节点相连的权值最小的边加入树中，并且将边所连的节点加入到树中。再找到与树中节点相连的最小权重的边，加入树，节点加入树，重复操作。

    算法的时间复杂度为$O(|V^2|)$算法并不依赖于边数$|E|$因此用于求解稀疏图非常合适。

2. Kruskal算法：与Prim算法的从顶点开始按照权重添加边和节点的方式不同，Kruskal算法是从边入手。刚开始将图G视为有V个树的森林，从权重最小的边开始选择，如果选择的边加入树之后构成了回路，那么就将这条边舍弃，按权重从小到大的顺序继续选择边，加入树。重复操作直到图G中只有一棵树，并且边数比节点树少1为止。

    算法的时间复杂度：由于Kruskal算法的边是通过堆来存储的，通过堆排序查找权重最小的边时间复杂度为$O(log|E|)$与Prim算法相反，Kruskal算法适合用来生成稀疏图的最小生成树。
### 最短路径问题

对于非带权图来说，广度优先搜索可以找到最短路径，带权图的路径长度是**带权路径长度**，对于带权图来说，我们要找到一个带权路径长度最小的路径，这条路径也就是最短路径。

#### Dijkstra算法求解单源最短路径问题

单源最短路径问题：图中某一顶点到其他顶点的最短路径。

设置一个集合用于记录已经求得的最短路径的顶点，在初始状态下，先将顶点$V_0$放入集合中，之后加入新的顶点$V_1$的时候需要需要更新从$v_0$到$V_1$的最短路径长度。在实际构建过程中需要两个辅助数组。
1. $dist[]$ 用于记录从原点到各点的最短路径长度。比如：当从$v_0$到$v_i$有弧时，将$dist[i]$的值更新为，从$v_0$到$v_i$的弧的权重，如果不连通，那么就讲数组的值记录为无穷。
2. $path[]$用于记录从原点到节点i的前驱，方便Dijkstra算法结束之后追溯最短路径。

求解过程如下：

首先我们初始节点放入集合中，并且将dist数组中的其他元素更新路径长度，从初始路径能够到达的节点我们就更新其权重值，如果无法到的那么就将其设为无穷。选择所有节点中路径最小的节点进行遍历，如果从初始节点到达的节点比从第二次选择的节点到达的节点权重值还要大的话，那么就将那个节点路径长度更新为最小的路径长度。再从dist数组中选择最小的节点，继续遍历。重复上述操作就可以将有向带权连通图的最短路径长度求解。

时间复杂度：对于邻接表和邻接矩阵来说，时间复杂度均为：$O(|V|^2)$。由于Dijkstra算法求解到某一顶点的最短路径，但是会将所有的路径都求解一遍，因此算法和求解全部路径一样复杂。

需要注意的是，对于负权重的情况Dijkstra算法无法求解。

#### Floyd算法求解各顶点之间的最短路径问题

首先构建一个$V*V$的方阵，Floyd算法就是通过迭代方阵求得最小的路径。方阵的行代表路径的起点，方阵的列代表路径的终点。比如$A[1][2]$是指，从节点1到节点2的路径，这条路径是中没有其他顶点。首先我们初始化方阵，对于有路径的节点将其权重填入到方阵中，对于没有路径的节点其权重可以设置为无穷。这个初始化的方阵称作$A_{-1}[i][j]$。之后我们进行第一次迭代，产生的方阵是$A_0[i][j]$，初始化的方阵意思是两个节点之间不允许通过其他节点，直接到达。第一次迭代的方阵意思是，节点可以通过0号节点中转，到达其他节点。因此方阵的第0行和第0列不用更改，因为第一次迭代的意思就是如果节点都从0号节点中转的话是否有可能减少路径长度，如果路径长度减少了那么就更新矩阵，如果没减少就维持矩阵的原本数值。第一行和第一列不变的原因是：第一行和第一列分别是以0为起点和以0为终点的路径，不需要通过0节点来中转。同样对角线上的元素也不需要更改，因为是从自身到自身。通过迭代所有顶点更新方阵就能得到从某一节点到另一节点的最小路径长度。

Floyd算法允许路径中有负权重，但是不允许负权重构成回路。对于带权无向图同样可以使用Floyd算法，因为带权无向图可以视为是具有相同权重的往返路径。

#### 算法比较

对于求单源最短路径的Dijkstra算法来说，时间复杂度为$O(|V|^2)$如果想实现和Floyd算法同样的效果，也就是求每个节点之间的最短路径长度可以对所有节点都使用一次Dijkstra算法，时间复杂度为$O(|V|^3)$。Floyd算法的时间复杂度同样也是$O(|V|^3)$但是Floyd算法代码紧凑，不包含复杂的数据结构。

### 利用有向无环图描述表达式

有向无环图：顾名思义，有向，但是没有环。简称DAG图

对于使用二叉树描述表达式的时候有一些相同的子式会重复出现，因为二叉树兄弟节点直接没有关系，因此我们可使用DAG来简化二叉树描述的表达式。

### 拓扑排序

AOV网：使用DAG表示一个工程的每个节点表示活动，其中每一条有向遍表示某个活动必须先于另一个活动，活动之间有传递性，也就是不能让一个节点指向其本身。

拓扑排序：当一个DAG满足下列条件的时候我们称为该图的一个拓扑排序。

1. 每个节点出现且仅出现一次。
2. 若顶点A在顶点B之前，那么就不存在一条路径从顶点B到顶点A。

#### AOV网进行拓扑排序的常用算法

在AOV网中找到一个没有前驱的节点，输出并且删除节点以及以它为起点的边。重复操作直到AOV网为空或者不存在无前驱的节点（表示AOV网中一定有环）。

由于每个顶点删除的时候还要同时删除以它为起点的边，当使用邻接表存储时，因时间复杂度为$O(|V|+|E|)$。当采用邻接矩阵存储的时候时间复杂度为$O(|V|^2)$。也可以通过深度优先搜索来进行拓扑排序。

---

逆拓扑排序：与拓扑排序相反，选择没有后继的节点输出，删除其入度边。重复操作指导AOV网为空。

注意：

1. 工程可以从入度为0，没有前驱的节点开始，这个节点表示之前的活动都已经完成。
2. 如果节点有多个直接后继，那么拓扑排序的结果通常不相同。 但当节点在一个线性有序序列中且节点的有唯一的前后驱关系，那么拓扑排序的结果就是唯一的。
3. 对于一般图来说，如果它的邻接矩阵是三角矩阵，那么就存在拓扑排序。

### 关键路径

AOE网：在带权有向图中，用顶点表示事件，用边表示活动，边上的权值表示该活动的开销，这个网就被称为**用边表示活动的网络**

AOE和AOV都是有向无环图，但是对于顶点和边的意义不同。在AOV中边是没有权值的，仅表示节点的前后次序，在AOE中边是有权值的。

性质：

1. 只有在某节点所代表的事件发生时，从该节点出发的活动才能开始。
2. 只有在进入某节点的所有活动（边）都结束时节点所代表的时间才能呢发生。
3. 在AOE网中，只有一个节点的入度为0，称为开始顶点（源点）。表示的意义是整个工程的起点。也只有一个出度为0，称为结束顶点（汇点）用于表示整个工程结束。

#### 事件$v_k$发生的最早时间$ve(k)$

是指从源点$v_1$开始到顶点$v_k$的最长路径长度，事件$v_k$的最早发生时间决定了所有从$v_k$开始的活动能够开工的最早活动时间。在计算的时候根据下列公式从前往后进行，并且根据拓扑排序的原则进行计算。

$ve(k)=Max\{ve(j)+Weight(v_j, v_k)\}$

#### 事件$v_k$发生的最迟时间$vl(k)$

是指在不推迟整个工程的前提下，事件$v_k$的最晚发生时间。计算公式如下。需要注意的是计算最迟发生时间从后往前算，需要增设一个拓扑排序的序列，完成之后从栈顶到栈底便是逆拓扑排序序列，输出即可。

$vl(k)=Min\{vl(j)-Weight(v_k, v_j)\}$

#### 活动$a_i$的最早开始时间$e(i)$

是指该活动的弧的起点事件的最早开始时间$e(i)=ve(k)$。

#### 活动$a_i$的最迟开始时间$l(i)$

活动弧终点所表示的最迟发生时间与该活动弧的时间之差。$l(i)=vl(j)-Weight(v_k, v_j)$

##### 一个活动$a_i$最迟开始时间$l(i)$和最早开始时间$e(i)$的差额$d(i)=l(i)-e(i)$

是指在不增加整个工程时间的情况下，活动$a_i$可以拖延的时间。如果一个活动的时间余量$d(i)$为0，那么就意味着活动必须如期完成，否则将拖延整个工程的完工时间，因此称这个活动$a_i$是**关键活动**。

#### 求解关键路径的算法步骤

1. 从源点出发，将源点最早发生时间设置为0，使用拓扑排序求得各顶点的最早发生事件。
2. 从汇点出发，令汇点的最早发生时间等于源点的最早发生时间，利用逆拓扑排序求得其余各顶点的最迟发生时间。
3. 求的各弧的最早最晚发生时间
4. 求AOE网中的所有互动的差额，找到所有差额为0的点就是关键路径构成的路径就是关键路径。

对于关键路径的一些注意问题

1. 关键路径上所有的点都是关键活动，因此缩短关键活动的差额就可以降低整个工程的时间，但是不能随意降低，如果降低到某一个程度，关键活动就会变成非关键活动。
2. 对于一个AOE网来说，可能会包含很多条关键路径，如果只降低其中一天关键路径上的点点时间并不能降低整个工程的时间，需要降低所有关键路径上的点才能达到缩短工期的效果。

# 查找

## 基本概念

1. 查找：在数据结合中找到满足某种数据元素的过程。分为查找成功和查找失败。
2. 查找表：用于查找的数据集合称为查找表。是由同一类元素组成的，一般有四种：1. 查找某个元素是否在查找表中。2. 查找特定元素的各种属性。3. 在查找表中插入一个元素。4. 在查找表中删除某个元素。
3. 静态查找表：不对查找表进行修改的查找表称为静态查找表。适合的方法有：顺序查找、折半查找、散列查找。
4. 动态查找表：当需要对查找表动态的添加或者删除元素这种查找表称为动态查找表。适合的方法：二叉排序树、散列查找、二叉平衡树、B树。
5. 关键字：在数据元素中，唯一标识某个元素数据项的值，基于关键字的查找所得的结果是唯一的。比如学生的学号就是关键字。
6. 平均查找长度（ASL）：一次查找的长度是指需要比较的关键字的长度。平均查找长度是是指所有查找次数的关键字比较的平均值。
$ASL=\sum_{i=1}^n P_iC_i$n是查找表的长度，$P_i$是指第i个元素在查找表中的概率，$C_i$是指第i个元素找到时需要比较多次数。

## 一般线性表的顺序查找

### 基本思想

就是从线性表的一端开始进行搜索，如果找到结果就查找成功，如果遍历整个表没有找到结果就查找失败。

$ASL_{成功}=\sum_{i=1}^nP_i(n-i+1)$
$ASL_{失败}=n+1$

通常来说，在查找表中的元素概率是不同的，如果能将元素按照概率从高到低进行排列，对于顺序查找来说也可以提高查找效率。

## 有序线性表的顺序查找

对于无序表来说，想要寻找一个不存在的元素一定会遍历整个查找表，最终会查找失败。但对于一个有序表来说，如果从小到大排列，查找到比待查找元素大的元素时匹配失败，就会直接结束查找，返回查找失败。这样有序表的顺序查找成功时与无序表的ASL基本相同，但是查找失败的情况ASL会小。

$ASL_{不成功}\sum_{j=1}^nq_j(l_j-1)=n/2+n/(n+1)$

相对于一般的无序表顺序查找，有序表改进了在查找失败的时候的ASL。但是仍然需要与折半查找进行区分。首先顺序查找和元素是否能够随机读取没有关系，也就是说，顺序查找可以适用于链表，也可以适用于顺序表。但是折半查找需要元素满足能够被随机读取，也就是，顺序查找时，元素是一个接着一个被遍历的，但是折半查找时元素是跳跃的被遍历的。

## 折半查找

又称为二分查找。适用情况又两个，一是有序，二是顺序表。也就是，能够被随机访问的有序表。

### 基本思想

将给定的元素的值与查找表进行比较，如果匹配则查找成功，如果比中间的元素小则对于从小到大排列的顺序表来说，到其前半部分进行查找，依然找到前半部分的中间元素进行比较。重复操作直到找到待查元素。

折半查找的过程可以用二叉树来进行描述，称为**判定树**树中的每一个节点都是一个元素，左侧的小于节点元素，右侧的大于节点元素。判定树一定是一颗平衡树。在等概率的情况下查找成功的平均查找长度为$ASL=1/n \sum_{i=1}^nl_i=(n+1)/nlog_2(n+1) \approx log_2(n+1)-1$

当有判定树的时候ASL也非常好求：当查找成功时层次乘本层的查找成功节点树将其相加，当查找失败时，失败节点层次与上次节点相同，也就是如果与第三层进行匹配失败，虽然在判定树会显示从第三层的节点延伸出一个失败节点，但是这个节点的层次属于第三层。

折半查找需要非常方便的找到中间元素的位置，因此要求线性表必须能够随机索引。同时还要求查找表是有序的。

## 分块查找

分块查找又称**索引顺序查找**拥有顺序查找和折半查找的优点，既有动态结构，可以快速查找。

### 基本思想

将查找表划分为若干子块，子块内部没有顺序，子块之间要有次序，可以表现为：若从小到大看的话，第一个子块中的所有元素都要小于第二个子块中最小的元素。也可以说是，第一个子块中最大的元素要小于第二个子块中最小的元素。据此来建立一张索引表，这张表记录了每一个子块中的最大的元素以及他们的第一个元素的位置。

查找的时候分为两部分:
1. 在索引表中确定待查记录所在的块。无论是顺序查找还是折半查找都可以。
2. 在子块中进行顺序搜索，找到待查元素。

分块索引的平均查找长度ASL为索引块查找和块内顺序查找长度之和。$ASL=L_I+L_S$

## 二叉排序树（BST）

特征：
1. 左子树非空时，左子树所有节点的值均小于根节点上的值。
2. 右子树非空时，右子树所有节点的值均大于根节点上的值。

当对BST进行中序遍历的时候会得到一个递增的序列。

### BST查找

沿着某个分支逐层向下比较，当BST非空的时候先与根节点进行比较，如果匹配则查找成功，如果小于根节点则到根节点右边进行查找，重复对根节点的操作。反之如果大于根节点则进行和小于根节点相反的比较方法。因此BST进行搜索的时候是一个递归的过程。

### BST插入

BST是一种动态树表，其结构是在查找过程中逐渐生成的。当需要查找的值不在BST中时，则进行插入，一开始BST是一个空树则直接进行插入，如果非空则需要进行比较，如果大于根节点则在其右侧进行插入，如果小于根节点则在其左侧进行插入。当查找失败的时候一定是在BST的左或者右叶子节点。

### BST删除

对于叶子节点来说，要进行删除的时候可以直接删除，并不影响BST的性质。但是如果节点拥有左右子叶的话需要按情况来定：如果左子叶为空，删除直接后继的时候将右子树移动到删除节点的位置即可。对于右子叶来说也是一样的。

### BST的查找效率

如果BST的叶子节点之间的高度绝对值差不大于1，那么这颗BST就是一个平衡二叉树，对于一棵平衡的二叉树来说，查找时间复杂度为$O(log_2n)$，最坏的情况为这颗BST是一棵倾斜的单枝树，那么查找的时间复杂度就会上升为$O(n)$，同时，BST的高度也会上升为元素的个数$n$。

### 与二分查找的比较

二者很相似，但是二分查找的判定树是唯一的，BST的判定树会根据插入的顺序不同而有所变化。至于二者的维护表的有序性来说。二分查找使用一个顺序表，而BST使用动态查找表，使用指针进行连接。

当有序表是静态查找表的时候应当使用顺序存储结构，查找方式采用二分查找。当使用动态查找表的时候，应采用链式存储结构，使用BST方式进行查找。

## 二叉平衡树

### 定义

为了避免BST高度增长速度过快，规定，在插入删除的时候要保证任意节点的左右子树的高度差不大于1，这样的树称为二叉平衡树。定义左右子树的高度差为该节点的平衡因子，因此平衡二叉树的平衡因子只能是-1、0、1。

### 二叉平衡树的插入

在插入的过程中首先要判断，插入是否会导致平衡因此大于$|1|$。如果导致了不平衡，那么就要在插入路径上找到最小的不平衡子树（插入路径上离插入节点最近的平衡因子绝对值大于1 的节点为根的树），并且对这个子树进行节点调节使其保持平衡。

#### 平衡二叉树插入时造成不平衡进行的调整

1. LL平衡旋转（右单旋转）：在节点A的左孩子的左子树插入节点，导致了以A为根的节点的平衡因子被破坏。这时候需要将A的左孩子节点B向右上移动代替A节点的位置，并且将A向右下移动成为B的右孩子节点，并且此时将B的原本的右孩子节点交给A的左孩子节点。这样就完成了LL平衡旋转。
2. RR平衡旋转（左单旋转）：在节点A的右孩子的右子树上插入节点导致节点A的平衡因子绝对值大于1，需要进行RR平衡旋转。具体方法与LL平衡旋转很类似，就是将节点A的右子树向左上方移动代替A的位置，并且使A成为左子树，将B节点原本的左子树交给A节点成为其右子树。这样就完成了RR平衡旋转。
3. LR平衡旋转（先左后右双旋转）：在A节点的左孩子B的右子树插入节点，导致A节点的平衡因子被破坏。如果B的右子树非空的时候需要将B的右子树的根节点C先左旋转提升到B的位置，再右旋转提升到A的位置，并使原本的A的右孩子交给节点C成为其右孩子，而B节点则成为C节点的左孩子。对于B的右子树为空的情况时，此时新节点就成为B节点右子树的根节点C，此时对C节点的操作就变成了对新节点的操作。
4. RL平衡旋转（先右后左双旋转）：在A节点的右孩子B的左子树节点插入新的节点导致节点A的平衡因子被破坏。将A节点的右孩子B的左子树根节点C提升到B节点的位置，再提升C节点到A节点的位置。之后时原本的A节点成为C节点的左子树，原本的B节点成为C节点的右子树，之后再进行新节点的插入。这是在A节点的右孩子的左子树非空的情况。当B节点的左子树为空的时候，此时新节点就是B节点左子树的根节点，将新节点提升到B节点的位置，再提升到A节点的位置，之后的操作与左子树非空的情况一致。

RL平衡旋转和LR平衡旋转在本质上是相同的，只不过对称进行操作。

### 平衡二叉树的删除

删除节点Z造成平衡二叉树被破坏时，找到最小不平衡子树的根节点Z，之后在找到Z节点的子树中高度最高的子树这棵子树的根节点为Y，再找到Y节点中高度最高的子树的根节点X。之后的调整有以下四种情况：

1. Y是Z的左孩子，X是Y的左孩子：进行LL平衡旋转。
2. Y是Z的左孩子，X是Y的右孩子：进行LR平衡旋转。
3. Y是Z的右孩子，X是Y的左孩子：进行RL平衡旋转。
4. Y是Z的右孩子，X是Y的右孩子：进行RR平衡旋转。

BBT的删除和BBT的插入基本相同，区别在于，在BBT的插入时只需要对以Z节点为根的树进行操作。但是在BBT的删除过程中，Z节点的平衡高度-1那么有可能也需要对Z的祖先节点进行调整，甚至追溯到整棵树的根节点，最终导致整棵树的高度-1。

### 平衡二叉树的查找

平衡二叉树是为了避免二叉排序树单枝造成查找效率低下进行的优化，因此，BBT的查找方式与BST相同。在查找过程中比较次数不会大于BBT的高度，因此BBT的$ASL=O(log_2n)$

## 红黑树

为了保持AVL的平衡性，在插入删除之后需要非常频繁的对AVL的拓扑结构进行调整，由此产生的效率低下，因此，引出红黑树的结构。

### 性质

当满足如下性质时的二叉排序树称为红黑树

1. 每个节点都是红色或者黑色的。
2. 根节点时黑色的。
3. 叶子节点（NULL指针、虚构的节点）都是黑色的。
4. 红节点的父结点和孩子节点都是黑色的，也就是不存在两个相邻的红节点。
5. 从某一节点到任一节点的简单路径上所包含的黑节点数目都是相同的。

### 红黑树插入

首先要树立一个概念，红黑树所有叶子节点都有一个虚构的节点是黑色的。并且其根节点之上也有一个虚构的父结点是黑色的，这些节点我们使用指针nil来表示。

根据情况的不同，红黑树的插入也不一样。首先我们默认插入的节点都是红色的，然后根据情况进行变色。可能的情况如下：

1. 当整棵红黑树都是空树的时候直接插入节点，并且变黑。
2. 当父结点是黑色的时候，这个时候直接插入即可。
3. 当父结点是红色的时候，这个时候要看叔节点（父结点的兄弟节点）。这里需要用到开头提到的虚构节点的概念。假设在某棵树上插入的时候其父结点没有兄弟节点，那么我们需要虚构一个节点，并且这个节点直接虚构为黑色的。
    当叔节点是黑色的时候，对插入节点、插入节点的父结点及其爷节点做平衡单旋转或者平衡双旋转，单双旋转取决于插入节点与父、爷节点是否处于同一直线上。父是爷的左，子是父的右要做双旋，父是爷的左，子也是爷的左就做平衡单选，这一点和AVL相同。这里还要注意，平衡双旋转应当拆开，也就是第一次旋转是将折线形转化为直线形，在做单旋转的时候要交换**原**父爷节点的颜色。
4. 当父结点和叔节点都是红的时候，根据红黑树的性质，父叔节点为红，其爷节点必为黑色。当父叔都是红色的时候，父叔节点变黑色色，爷节点变红色。因为改变了爷节点的颜色，很有可能会破坏上一层的红黑树的性质，因此要逐两层向上寻找判断是否应该变色。

总结：只有在父结点为红色的时候才需要对叔节点进行判断。父红叔黑这种情况既需要做平衡旋转，也需要变色（无论单双旋转都需要变色）双旋只是将折线形转化为直线形，再进行单选，变色过程是在单旋转中产生的。之外，其他情况只需要变色。

## 直接插入排序

### 基本思路

将乱序序列分为三部分，第一部分从[1-i]为有序序列，第二部分为第i个元素。第三部分为[i-n]位无序序列。每次都将i与第一部分的元素进行比较找到插入的位置。将有序序列中插入位置之后的元素后移一位。将i复制到正确位置。

### 时间复杂度

每一趟都分为比较关键字和插入元素，共进行了n-1次。比较和插入的复杂度由待排序列的有序程度决定。

最好情况为正序排列，只进行一次比较时间复杂度为$O(n)$，插入的时间复杂度为$O(1)$

最坏情况为逆序排列，比较的时间复杂度为$\sum_{i=2}^ni$插入的时间复杂度为$\sum_{i=2}^n(i+1)$

平均时间复杂度为$O(n^2)$

### 空间复杂度

需要一个辅助单元进行交换，因此空间复杂度为$O(1)$

### 稳定性

每趟都是从后向前进行比较，不会改变相对位置。因此直接插入排序是稳定的算法。

## 折半插入排序

### 基本思路

由于直接插入排序前半部分为有序序列，可以通过二分查找法确定需要插入的位置。因此可以通过改进直接插入排序的比较部分降低算法的时间复杂度。

### 时间复杂度

通过折半查找降低了比较的时间复杂度为$O(nlog_2n)$，但是并未降低插入的时间复杂度，因此总体的时间复杂度仍为$O(n^2)$。但当数据量不大的时候折半查找能够表现出很好的性能。

### 空间复杂度

与直接插入排序相同，只需要一个辅助空间进行交换。因此时间复杂度仍为$O(1)$

### 稳定性

通过改进直接插入排序，因此折半插入排序也是稳定的。

## 希尔排序

### 基本思路

当直接插入排序序列就是有序序列时，时间复杂度会降低到$O(n)$，因此我们可以通过将待排序序例基本有序之后再进行直接插入排序。

我们通过选取一个步长 $d<n$ ，将每间隔d-1的元素组成一个序列，分别对这些序列进行直接插入排序。完成后取小于第一次对d再次进行操作。直到d=1时。

此时再对整个序列进行直接插入排序。

### 时间复杂度

由于d的取值也就是增量序列的函数是数学上一个未解决的问题。时间复杂度比较难分析，最坏情况下的时间复杂度为$O(n^2)$

### 空间复杂度

与直接插入排序相同，需要1个辅助空间进行交换，因此空间复杂度为$O(1)$

### 稳定性

由于相同的元素被分配到不同的子表中，其前后位置会发生改变，因此希尔排序是不稳定的。

交换排序

## 冒泡排序

### 基本思路

从后往前俩俩比较，若为逆序则交换位置，直到序列比较完成。每一趟都有一个元素到达正确的位置。最多进行$n-1$趟排序就能完成。当某趟排序没有发生交换的时候证明序列已经有序。

### 时间复杂度

最好的情况是第一次没有进行交换，此时序列有序，但是仍然要比较n-1次，此时的时间复杂度为$O(n)$

当发生交换的时候每次交换都要移动元素三次。

最坏和平均时间复杂度均为$O(n^2)$

### 空间复杂度

需要常数个辅助单元用来进行交换，因此空间复杂度为$O(1)$

### 稳定性

稳定

## 快速排序

### 基本思路

在开头指定一个pivot，并且指定两个指针，首先low在最右边从右向左搜索比pivot小的元素，找到后将low指向的元素复制到high处。之后high从左向右搜索比pivot大的元素，将high复制到low的位置。当两个指针相遇之后将pivot的值复制指针处。

### 时间复杂度

快速排序的运行时间与pivot将两部分划分的是否对称有关，越对称递归的层数越小。

最坏情况：当待排序序列是有序活着逆序序列当时候，pivot每次都是最边缘的位置，也就是将两部分划分的极其不均匀。这个时候快排效率最差，甚至比直接插入排序的效率还要差时间复杂度为$O(n^2)$。

最好情况：两部分分配十分均匀，此时的时间复杂度大大提升为$O(nlog_2n)$快速排序的平均情况接近最好情况

快速排序算法是所有内部排序算法中平均性能最优的算法。

### 对快排算法进行改进

1. 当待排元素为5～25个的时候快排的效率比较差，在大量数据排序的时候一定会有这个过程，但是我们可以通过快排加直接插入的方式，一开始数据很多的时候使用快排，当每一组数据剩余很少的时候可以使用直接插入完成排序。
2. 我们还可以通过先使用快速排序将待排序列整体有序，之后再使用直接插入排序，因为直接插入的算法效率是跟待排序例的有序程度有关系的。
3. 由于快排的效率取决于选取的pivot元素在整个待排序列中的位置（pivot越能将待排序列均匀分为两份快排越高效），因此我们可以选取开头节点、结尾节点和中间节点，找出其中位数来确定pivot，可以极大降低pivot选取的不合理的问题。

### 空间复杂度

每次划分都需要通过栈进行调用。最坏情况是下需要调用$n-1$次平均情况下栈的深度为$O(log_2n)$

### 稳定性

pivot左侧的值均为小于pivot的值，因此在交换值的过程中会产生位置变化。快排是不稳定的算法。

## 选择排序

### 基本思路

将一个乱序序列从头到尾找到一个最小的，将最小的放到头部，此时头部的元素已经排序完成。在第i次排序中从[i..n]中选择最小的与i进行交换。

### 时间复杂度

元素的移动次数：不会超过$3(n-1)$，最理想的情况下移动0次此时元素已经有序。

元素的比较次数：与序列的初始状态无关，因此始终是$n(n-1)/2$次。

因此时间复杂度始终是$O(n^2)$

### 空间复杂度

需要常数个辅助单元进行交换，因此空间复杂度为$O(1)$

### 稳定性

因为与i进行交换所以可能会发生位置的变动。因此算法不稳定。

## 堆排序

大根堆：堆顶元素为最大值。

小根堆：堆顶元素为最小值。

### 基本思路

通过构建大根堆或者小根堆可以将元素有大到小或者由小到大进行排序。首先将乱序序列构建为大根堆，输出堆定元素，之后将被破坏的堆重新构建为大根堆。重复进行。

以下都是根据大根堆进行说明。

---

要点:

1. 如何将乱序序列构建为大根堆
2. 输出堆顶元素后，堆被破坏，如何进行调整。

### 堆的初始化

自下而上

将一个乱序序列一次加入堆中，若新加入的节点比它的根节点要大则与根节点交换位置。

### 堆的调整

自上而下

当输出堆顶元素之后，大根堆被破坏，此时我们需要将最下层最后一个叶子节点放到堆定，并自上而下将被破坏的堆调整为大根堆。

通过二叉树定义。子节点与父结点有直接关系，跟兄弟节点无关系。

## 归并排序

### 基本思想

乱序序列中有n个元素，可将其视为n个c长度为1的有序序列。其中两两归并为一个有序序列，如此重复直到变为一个长度为n的有序序列，排序完成。

### 时间复杂度

每趟归并的时间复杂度为$O(n)$总共需要进行$\lceil log_2n\rceil$次。因此算法的时间复杂度为$O(nlog_2n)$

### 空间复杂度

在归并过程中，每次都需要n个辅助空间，因此算法的空间复杂度刚好是$O(n)$

### 稳定性

每次都是按照顺序进行归并，不会改变相同元素的位序，因此是稳定的。

## 基数排序

### 基本思想

最高位优先（MSD）：将关键字按照权重递减，最高位先进行排序。

最低位优先（LSD）：与MSD相反，最低位先进性排序，直到最高位。

基数排序不依靠比较和移动进行排序，根据关键字各位的大小进行排序。以LSD为例，首先创建10个从0-9的队列，从最低位个位开始，按照乱序队列的顺序依次将元素连接到辅助队列末尾。然后进行从上到下从左到右进行收集。完成收集之后再进行十位的分配与收集。重复，直到分配收集到最高位。

### 时间复杂度

基数排序需要最高位位数d次分配和收集，每次分配都要进行待排元素个数n次，每次收集都需要队列个数r次。因此时间复杂度为$O(d(n+r))$

### 空间复杂度

分配时需要r个辅助队列，每个队列包含r个队头指针和r个队尾指针，但是这些队列可以重复使用，因此空间复杂度为$O(r)$

### 稳定性

基数排序是依靠按位排序时的稳定性进行排序的，因此基数排序一定是稳定的。

### 关于为何使用LSD进行排序

首先位数越高对大小的影响也就越大，如果我们先按照高位进行排序，高位排好序，但是接下来进行的低位排序可能又会将高位打乱，因此我们要先将对大小影响较轻的低位排好序，再进行高位的排序。
