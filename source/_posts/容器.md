# 顺序容器

1. vector: 动态数组，动态分配内存，支持随机访问，除尾部外的所有位置插入删除效率很低。
2. deque: 双端对立，动态分配内存，支持随机访问，除首尾外的所有位置插入删除效率很低。
3. list: 双向链表，动态分配内存，不支持随机访问，在任何位置插入删除效率都很高。
4. forward_list: 单向链表，不支持size操作（其保存计算比手写链表多出额外开销。对于其他容器而言size操作是一个常量级的操作。），新标准引入，设计目的是达到与手写单向链表相同性能。
5. array: 固定数组大小，支持随机访问，不能添加删除元素，array很多操作都是受限的，并且实例化方式与其他容器不同，需要指定容器的大小，也就是无法动态分配内存。

## 选择容器的基本原则

1. 除非由更好的理由，否则使用vector。
2. 如果由很多小元素，并且空间开销敏感的话，不要使用list或forward_list。
3. 随机访问时应使用vector或deque。
4. 如要在中间插入删除元素，应使用list或forward_list。
5. 在插入时需要向中间插入元素，但之后读取时需要随机访问，可以先使用list，再转为vector。或者使用sort函数进行重排。
6. 如果既要在中间插入删除元素，又需要随机访问，需要进行测试，占主导地位的操作决定了使用哪种容器。

## vector

### 特点

1. 动态空间，随着元素加入空间会自动扩充，当由新元素插入时，如果空间足够会直接插入，如果空间不够会扩容至两倍或1.5倍，如果两倍不够，则扩容至足够大的空间。
2. 扩充过程并不是在原有空间后追加，而是重新申请一块新的连续内存，所以只要发生了内存空间的重新配置，所有迭代器都会失效。
3. 在内存中是连续的线性空间，普通指针可以作为vector的迭代器。
4. 由三个迭代器，指向头部的iterator，指向尾部的iterator和指向可用空间的iterator。

### 底层

采用动态数组

### 内存增长机制

为了能够支持快速随机访问，vector在内存中是一片连续的空间，但是由于vector需要能够动态的调整大小，假如在内存中vector后面的内存并不是空的，此时要天机元素，vector为了保持元素在内存中的连续性会将整个vector移动到一片更大的内从中去。

当`vec.push_back(val)`空间不够时，会自动申请另一块更大的内存空间，2倍（GCC），然后把原空间的数据拷贝到新空间，释放原空间。

当vec.claer()或元素被释放时，vec内存不释放，仅清空数据。

## 容器适配器

适配器是一种机制，能够使某种事物的行为看起来像另外一种事物一样。比如说有一个stack适配器，其底层是deque，但是可以使用其他的容器作为其底层，这就是使某种事物看起来像另一种事物。

只要满足适配器的要求，就可以作为其底层构成适配器。所有的容器都是基于底层容器定制了自己的操作，也只能使用自己的操作，不能使用底层容器的操作。

由于所有的适配器都要求容器具有添加和删除元素的能力，因此，array无法作为适配器的底层。

一下是各种适配器的要求: 

1. stack: 默认使用deque来实现。只要求push_back, pop_back和back操作，因此除了array和forward_list之外的所有容器均可作为stack的底层。
2. queue: 默认使用deque来实现。要求back push_back front push_front。因此可以构造于list和deque，但是不能使用vector。
3. priority_queue: 要求front push_back pop_back还需要能够随机访问，因此可以使用vector和deque。

队列是一种先进先出（FIFO）的数据结构，而优先级队列priority_queue是优先级高的元素即使比优先级低的元素后进队列，也排在前面。

## 使用内置数组初始化顺序容器

标准库中的顺序容器有begin和end成员函数，分别可以获得该容器对应的开始元素的迭代器以及尾后迭代器。虽然内置数组没有这两个成员函数，但是在标准库中有两个函数可以得到内置数组首元素的指针以及尾后元素的指针。

对于数组来说，vector和string容器的迭代器所支持的运算数组的指针全部都支持。

```cpp
int a[] = {1, 2, 3}
void init_vec() {
    std::vector<int> vec;
    for(int* i = std::begin(a); i != std::end(a); i++) {
        vec.push_back(*i);
    }
}

void init_list() {
    std::list<int> list;
    for(auto i = std::begin(a); i != std::end(a); i++) {
        list.emplace_front(*i);
    }
}
```

除了手动构建的方法之外可以使用泛型算法和插入迭代器的方式来构建: 

```cpp
void copy_array_to_vector() {
    int arr[] = {1, 2, 3, 4, 5, 6, 7, 8};
    std::vector<int> vec;
    std::copy(std::begin(arr), std::end(arr), std::back_inserter(vec));
    std::copy(vec.cbegin(), vec.cend(), std::ostream_iterator<int> (std::cout, " "));
}
```

首先通过列表初始化了一个内置数组类型，元素类型为int，接着创建了一个空的vector容器，下面的是重点: 首先调用了泛型算法的copy算法，将arr的元素使用插入迭代器插入到了vec中，接着又使用同样的算法将vec中的元素复制到输出流迭代器中，这里并非在外面初始化一个对象，而是直接在传入算法的参数中直接初始化了一个输出流迭代器。输出流迭代器的详细内容可以看[迭代器](../_posts/迭代器.md)部分的内容。

# 关联容器

关联容器与元素加入容器的顺序无关，能够高效的进行关键字查找。

1. map: 当中的元素是一个键值对，通过对键进行索引，值是作为键的相关数据。
2. set: 每个元素中只包含一个关键字，可以使用set高效的检查元素是否在其中。

八个关联容器可以通过三种不同的维度来进行分类: 

1. 是map还是set。
2. 关键字是否可以重复。multi
3. 是否按照加入元素的顺序保存。unordered_

## map

定义一个map时必须要指定键值元素类型。对于一个map来说，下标运算符会创建一个新的元素，如下word_count会创建一个新的元素，并且初始化为0，不论元素是否能创建都会将其值加1。

在map中每个元素都是一个pair类型的对象，pair是一个模板类型，保存first和second两个共有成员，分别对应着每个元素的键和值。

查找可以使用find成员函数进行查找，对于map如果通过键找到了pair则返回该位置上的迭代器，对于set同样返回指向该元素的迭代器。如果没有找到则返回该容器的end迭代器，因此可以通过返回的迭代器与end迭代器比较是否相等来判断元素是否存在于容器中。

```cpp
std::map<std::string, size_t> word_count;
std::string word;
while(std::cin >> word) {
    word_count[word]++;
    std::cout << word_count.find(word)->first << std::endl;
    std::cout << word_count.find(word)->second << std::endl;
}

```
