泛型算法不会直接操作容器，而是运行在迭代器之上，因此泛型算法蕴含了一个编程假定: 算法永远不会改变底层容器的大小，可能改变容器中保存的元素的值，可能在容器中移动元素，但是不会直接添加或者删除元素。

# lambda表达式

在C++中有四种可调用对象: 

1. 函数
2. 函数指针
3. 重载了函数调用运算符的类
4. lambda表达式

一个lambda表达式表示一个可以调用的代码单元，可以理解为未命名的内联函数。表达式的形式如下: 

```cpp
[]() -> void {
    // 表达式的内容
}
```

开头的中括号表示捕获列表，圆括号是表达式的参数，对于lambda表达式来说返回类型必须是尾置的，接下来就是函数体。一个lambda表达式捕获列表和函数体是必须的，圆括号和尾置返回类型可以适当省略。

对于lambda的返回类型来说还有一点需要注意: 如果在lambda表达式中有除了`return`之外的语句，那么在不显示声明表达式类型的时候会返回void，当只有一条`return`语句的时候lambda会根据返回语句推断出返回值的类型。

## 捕获列表

如果我们在一个函数中创建了一个lambda表达式，又想使用函数中的局部变量，lambda表达式无法直接使用，必须要将这些局部变量通过捕获列表传入lambda表达式。需要注意: 捕获列表是在lambda创建的时候传入的。lambda表达式创建和调用可以中间相隔一段代码，具体如下所示: 

```cpp
void test_lambda() {
    auto f = []() {
        return {42;}
    };

    // 中间处理的代码

    f();
}
```

当然直接在函数的参数中传递一个lambda表达式会创建后立即执行。

因此我们会遇到一个问题，首先，对于捕获列表来说，捕获有两种形式: 

1. 值捕获: 这种捕获是将捕获的值复制进lambda表达式中，当然前提是该变量是可以拷贝的，比如对于无法拷贝的IO对象就无法使用值捕获。又因为值被拷贝进表达式，因此后续改变了值也不会影响到lambda中对应的值。
2. 引用捕获: 与引用传参类似，着重考虑引用需要注意的问题，在lambda执行期间引用要一直存在，捕获的局部变量在函数结束之后就不复存在了，可以从一个函数中返回一个lambda表达式，由于函数无法返回一个局部变量的引用，虽然函数可以返回一个lambda表达式，但是基于上面的道理，lambda表达式不能捕获引用。
